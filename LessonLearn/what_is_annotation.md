# 型別、註解、與「annotation」在說什麼

## 為什麼會聽到「註解」這個字眼？不是應該說「型態／型別」嗎？

這主要是**用詞習慣**，不是說「型別」錯了。

### 英文裡其實是兩個概念

- **Type（型別）**：`str`、`int`、`AIMessage` 這些**型別本身**。
- **Annotation（註解／標註）**：在程式裡**寫在參數或回傳值旁邊的那一段標記**，例如 `def f(x: int) -> str` 裡的 `: int` 和 `-> str`。

所以英文常說 **type annotations**（型別註解、型別標註），指的是「**把型別資訊標在程式碼上的寫法**」，不是指你用 `#` 寫的一般註解（comment）。

### 中文怎麼說比較清楚

| 說法 | 通常指什麼 |
|------|------------|
| **型別** | `str`、`list[Any]` 這些型別名稱本身 |
| **型別註解／型別標註** | `x: str`、`-> AIMessage` 這整段「標註行為」 |
| **註解**（單獨講時） | 常被聯想成 `# 這行是註解`，容易混淆 |

用「註解」是沿用 **annotation → 註解** 的翻譯習慣；若覺得容易和 `#` 搞混，**改說「型別標註」或「型別註解」**會更精準。

### 一句話整理

**「型態／型別」是內容，「註解／標註」是寫在程式上的那種標記方式**；兩個詞講的是同一件事的兩個面向，不是互相排斥。

## 補充：`from __future__ import annotations` 和這有什麼關係？

在 Python 3.7+ 可啟用 **PEP 563**：型別註解會以**字串形式延後解析**（lazy evaluation）。

常見用途：

1. **前向引用**：類別方法回傳「自己的類型」時，有時不必加引號。
2. **減少載入模組時立刻解析複雜型別** 的負擔。

若程式裡沒有前向引用、也沒有特別需求，**不必**為了「型別標註」而強行加上這一行；有無此行，對簡單腳本往往行為相同。
